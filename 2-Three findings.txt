● Three findings:

  theory_kappa_min is not broken — values are 1e-7 to 1e-13, genuinely near-zero. The MLP display showed 0.0000 due to
  float rounding in the summary. Some ZCTAs are very "easy" relative to the LOO reference set — their prediction error
  is essentially at the minimum achievable, so D*/D → 0 for those samples.

  Simplex integrity fails for 96/315 records (30.5%). Pattern is nearly pure:
  - Overwhelmingly elevation task
  - All solvers/embeddings affected, but ridge + elevation is catastrophic (sum as low as 0.615)
  - Cause: the MLP residual classifier degenerated on elevation — the 3-class (easy/medium/hard) boundaries broke down
  because elevation has extreme geographic gradients that don't form clean terciles. The MLP learned to ignore one
  class, so probability mass was lost (R+S+N < 1). This contaminates rsct_qual (which depends on R and N) for those
  cells, but rsct_compat is computed independently from the MLP so it's clean.

  omega is nearly constant (mean=0.664, std=0.015) — the known V-001 governance gap. It's a hardcoded proxy, not a
  learned distributional reliability estimate.

  What this means for the turbulence analysis: the rsct_turb and rsct_compat results are valid — both are
  MLP-independent. But any analysis using rsct_qual, alpha, R, or N should flag the 30.5% simplex violation as a
  confounder, especially for the elevation task where virtually all records are affected.
